# 届次提取逻辑优化计划

## 一、需求概述

将届次(JC)的获取方式从"文件名解析"改为"AI提取"，并修复年度股东大会查询时JC为空无法匹配的bug。

## 二、当前问题分析

### 2.1 现有流程

```
文件名 → _extract_basic_info() → basic_data(含meeting_session)
                                    ↓
                        process_file_with_uploaded_id_bjqk/hbbj()
                                    ↓
                        get_data_from_db_bj/hb(**basic_data)
                                    ↓
                        SQL: ... AND A.JC = ? (直接传参)
```

### 2.2 存在的问题

1. **届次来源不可靠**：依赖文件名格式规范，可能解析失败
2. **年度股东大会查询bug**：当JC为空字符串时，SQL使用 `JC = ''` 无法匹配数据库中的 `NULL` 值

### 2.3 AI返回的届次结构

**表决情况(prompt\_GDDHBJ.md)**：

```json
{
  "届次": "1",           // 临时股东大会
  "extracted_data": [...]
}
```

或

```json
{
  "届次": "",            // 年度股东大会（空字符串）
  "extracted_data": [...]
}
```

**出席回避(prompt\_GDDHCX\_HBBJ.md)**：

```json
{
  "extracted_data": {
    "届次": "1",
    "basic_data": {...},
    "avoid_data": [...]
  }
}
```

## 三、优化方案

### 3.1 整体流程调整

```
文件名 → _extract_basic_info() → basic_data(备用)
                                    ↓
                        process_file_with_uploaded_id_bjqk/hbbj()
                                    ↓
                        AI提取 → 从AI结果获取届次
                                    ↓
                        届次为空? → 使用basic_data.meeting_session备用
                                    ↓
                        get_data_from_db_xxx(动态SQL)
                                    ↓
                        届次为空? → SQL: JC IS NULL
                        届次有值? → SQL: JC = ?
```

### 3.2 修改清单

#### 修改1：`process_file_with_uploaded_id_bjqk` 函数 (约L282-340)

**修改内容**：

* 新增从AI提取结果中获取届次的逻辑

* AI届次为空时使用basic\_data.meeting\_session作为备用

* 更新basic\_data中的meeting\_session值

**修改前**：

```python
def process_file_with_uploaded_id_bjqk(self, file_id: str, filename: str, basic_data):
    extracted_data = enhanced_ai_service.extract_data_from_file(...)
    # 直接使用basic_data查询
    sql_data = self.get_data_from_db_bj(**basic_data)
```

**修改后**：

```python
def process_file_with_uploaded_id_bjqk(self, file_id: str, filename: str, basic_data):
    extracted_data = enhanced_ai_service.extract_data_from_file(...)
    # 从AI结果提取届次
    ai_jc = extracted_data.get("届次", "")
    # 更新basic_data中的届次（AI优先，文件名备用）
    basic_data["meeting_session"] = ai_jc if ai_jc != "" else basic_data.get("meeting_session")
    # 使用动态SQL查询
    sql_data = self.get_data_from_db_bj(**basic_data)
```

#### 修改2：`process_file_with_uploaded_id_hbbj` 函数 (约L342-410)

**修改内容**：同上，从AI结果中提取届次

**AI返回结构差异**：

* 表决情况：`{"届次": "1", "extracted_data": [...]}`

* 出席回避：`{"extracted_data": {"届次": "1", "basic_data": {...}, "avoid_data": [...]}}`

#### 修改3：`get_data_from_db_bj` 函数 (约L412-421)

**修改内容**：改为动态SQL构建

**修改前**：

```python
def get_data_from_db_bj(self, stock_code, info_date, meeting_type, meeting_session):
    return db_manager.execute_query(
        SQL_QUERY_BJ,
        (stock_code, info_date, meeting_type, meeting_session)
    )
```

**修改后**：

```python
def get_data_from_db_bj(self, stock_code, info_date, meeting_type, meeting_session):
    # 动态构建SQL
    if meeting_session and str(meeting_session).strip():
        sql = SQL_QUERY_BJ.replace("AND A.JC = ?", "AND A.JC = ?")
        params = (stock_code, info_date, meeting_type, meeting_session)
    else:
        # 年度股东大会：JC为空，使用IS NULL
        sql = SQL_QUERY_BJ.replace("AND A.JC = ?", "AND A.JC IS NULL")
        params = (stock_code, info_date, meeting_type)
    return db_manager.execute_query(sql, params)
```

#### 修改4：`get_data_from_db_hb` 函数 (约L423-434)

**修改内容**：同上，改为动态SQL构建

#### 修改5：保留`_extract_basic_info`函数作为备用

**修改内容**：保留现有逻辑，但注释说明届次优先使用AI提取结果

## 五、测试场景

| 场景        | AI届次 | 文件名届次 | 预期SQL        | 预期结果    |
| --------- | ---- | ----- | ------------ | ------- |
| 临时股东大会第1次 | "1"  | "1"   | `JC = '1'`   | 正常查询    |
| 临时股东大会第2次 | "2"  | "1"   | `JC = '2'`   | 使用AI值   |
| 年度股东大会    | ""   | "1"   | `JC IS NULL` | 正常查询    |
| AI提取失败    | None | "1"   | `JC = '1'`   | 使用文件名备用 |
| AI提取失败且年度 | None | None  | `JC IS NULL` | 正常查询    |

## 六、风险评估

| 风险           | 影响     | 缓解措施                    |
| ------------ | ------ | ----------------------- |
| AI返回格式变化     | 届次提取失败 | 使用文件名备用方案               |
| 数据库JC字段非NULL | 查询无结果  | 需确认数据库年度股东大会JC字段是否为NULL |
| 动态SQL注入风险    | 安全问题   | 参数仍使用参数化查询，仅动态替换SQL片段   |
