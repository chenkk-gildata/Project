# 金融数据AI抽取提示词优化分析报告

## 一、当前提示词优点分析

### 1. 结构设计合理

* 分章节阐述：字段定义 → 数据来源处理 → 数据处理 → 输出要求

* 层次分明，便于AI理解任务流程

### 2. 字段定义详细

* 包含32个字段的完整定义

* 每字段都有匹配关键字、不包含关键字、业务描述

* 覆盖了上市公司年报的主要财务指标

### 3. 案例说明丰富

* 包含正确和错误案例对比

* 对净资产收益率的特殊处理有详细说明

### 4. 核心原则明确

* 原子化映射原则

* 禁止计算原则

* 空值处理规则

## 二、存在的问题与优化建议

### 问题1：字段数量前后不一致

**问题描述：**

* 字段定义表有32个字段

* 核心原则提到"确保所有29个字段都检查过"

**优化建议：**

* 统一字段数量为32个

* 在核心原则中明确"确保所有32个字段都检查过"

***

### 问题2：正则匹配规则不够精确

**问题描述：**

* 第15行 `JBMGSY`: "完全等于'基本每股收益' OR '每股收益'"

  * 问题："每股收益"范围太宽，可能匹配到"稀释每股收益"

* 第29行 `JLR`: `'归.*净(利润|亏损)'`

  * 问题：可能匹配到"归属于普通股股东的净利润"（这是PTGJLR）

**优化建议：**

```markdown
# 修改JBMGSY匹配规则
JBMGSY: 完全等于"基本每股收益" 或以"基本每股收益"开头

# 修改JLR匹配规则  
JLR: '归.*净(利润|亏损)' 且不匹配'普通股'
```

***

### 问题3：字段定义表正则表达式表述不当

**问题描述：**

* 第19行："包含'净资产收益率' AND '加权'"

* 问题：正则表达式中"包含"和"AND"不是标准正则语法

**优化建议：**

* 使用标准正则表达式语法

* 例如：`'净资产收益率.*加权|加权.*净资产收益率'`

***

### 问题4：部分字段定位不明确

**问题描述：**

* FJCXSY字段只写"非经常性损益项目"

* 问题：年报中多处可能出现"非经常性损益"，需要更精确定位

**优化建议：**

```markdown
# FJCXSY字段优化
定位：主要会计数据和财务指标表格中的"非经常性损益"行
关键字优先级：
  1. "影响母公司股东净利润的非经常性损益"
  2. "非经常性损益净额" 
  3. "合计"
排除：明细表格中的数据
```

***

### 问题5：缺少季度报和年报的区分说明

**问题描述：**

* 提示词只提到年度报告

* 实际业务可能需要处理季度报告

**优化建议：**
在JZRQ字段处理规则中添加：

```markdown
# 报告期类型区分
- 年度报告：截止日期为'20XX-12-31'
- 季度报告：截止日期为'20XX-03-31'、'20XX-06-30'、'20XX-09-30'
- 季度报告只披露本季度和本年累计数据，不披露上年同期
```

***

### 问题6：字段名笔误

**问题描述：**

* 第74行："禁止将'营业收入(YYSR)'的数据作为'营业总收入(YYZSRS)'的数据输出"

* 字段名写错了：YYZSRS → YYZSR

**优化建议：**

* 修正为正确的字段名

***

### 问题7：缺少多表格优先级说明

**问题描述：**

* 当"主要会计数据和财务指标"和"净资产收益率及每股收益"两个表格都有数据时，没有说明优先级

**优化建议：**

```markdown
# 多数据源优先级
每股收益数据：
  1. "净资产收益率及每股收益"表格（数据更精确）
  2. "主要会计数据和财务指标"表格

净资产收益率数据：
  1. "主要会计数据和财务指标"表格（普通收益率）
  2. "净资产收益率及每股收益"表格（普通股收益率）
```

***

### 问题8：缺少对负数格式的统一处理规则

**问题描述：**

* 第159行提到"括号处理"，但说明较简略

* 实际年报中负数可能有多种格式：(100)、-100、亏损100

**优化建议：**

```markdown
# 负数格式处理规则
- 括号表示法：(1,234.56) → -1234.56
- 亏损文字："亏损100万" → -1000000
- 负号表示法：保持原样
```

***

### 问题9：同比字段关联规则可更明确

**问题描述：**

* 第155-157行有字段关联规则，但不够细化

* 没有说明如果同比值为"减少XX%"（负数）时如何处理

**优化建议：**

```markdown
# 同比字段处理细则
- 同比为"增加"：转为正数
- 同比为"减少"：转为负数
- 同比为"持平"或"-"：为空
- 金额为空时，同比字段必须为空
```

***

### 问题10：部分字段定义重复/冗余

**问题描述：**

* ZCZE字段：匹配关键字"总资产金额"，不包含关键字也是"总资产金额"

* 这是重复的

**优化建议：**

```markdown
# ZCZE字段优化
匹配关键字：'总资产'
不包含关键字：'归属于'（避免匹配到归母资产）
```
