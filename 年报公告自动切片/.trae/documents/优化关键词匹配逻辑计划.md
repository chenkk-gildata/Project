# 优化关键词匹配逻辑计划

## 问题描述

当前代码在匹配关键词时存在bug：当有多个keyword时，代码会按keyword顺序遍历，找到第一个匹配的就返回。但如果同一页中同时存在多个keyword且都能匹配pattern时，应该返回位置最靠前的那个，而不是先遍历到的keyword的匹配。

### 问题示例

```
## 3、董事、高级管理人员薪酬情况     ← 用户希望匹配这个（位置靠前）
……
### 公司报告期内董事、监事和高级管理人员报酬情况   ← 实际匹配到这个（因为先遍历"报酬情况"）
```

## 需要优化的位置

代码中有4处关键词搜索逻辑需要优化：

### 1. 开始关键词搜索 (第100-113行)

* keyword列表: `["高级管理人员"]` - 当前只有1个keyword

* 需要优化以支持未来扩展

### 2. 结束关键词搜索 (第116-130行)

* keyword列表: `end_keyword`

  * 北交所: `["变动情况"]`

  * 上交所: `["变动情况"]`

  * 深交所: `["履行职责", "履职", "有关情况"]` - 3个keyword

### 3. 移除开始关键词搜索 (第134-148行)

* keyword列表: `remove_start_keyword`

  * 北交所: `[]`

  * 上交所: `["主要工作经历"]`

  * 深交所: `["高级管理人员离任"]`

### 4. 移除结束关键词搜索 (第150-164行)

* keyword列表: `remove_end_keyword`

  * 北交所: `[]`

  * 上交所: `["报酬情况", "薪酬情况"]` - 2个keyword

  * 深交所: `["报酬情况", "薪酬情况"]` - 2个keyword ← 用户举例的bug位置

## PDF库和坐标系说明

### 使用的PDF库

代码使用的是 **PyMuPDF (fitz)** 库进行关键词搜索：

```python
import fitz
instances = page.search_for(keyword)  # 返回 Rect 对象列表
```

### PyMuPDF坐标系

根据PyMuPDF官方文档：

* **原点 (0,0) 位于页面左上角**

* **X轴向右递增**（x值越大，位置越靠右）

* **Y轴向下递增**（y值越大，位置越靠下）

### Rect对象属性

`page.search_for(keyword)` 返回的是 `Rect` 对象列表，每个Rect包含：

* `x0`: 矩形左边界（较小值）

* `y0`: 矩形上边界（较小值，位置靠上）

* `x1`: 矩形右边界（较大值）

* `y1`: 矩形下边界（较大值，位置靠下）

### 位置排序逻辑

根据坐标系特性：

* **正常页面 (rotation=0)**：`y0` 值越小，位置越靠上，应该排在前面

* **旋转90度页面 (rotation=90)**：需要根据旋转后的布局调整，`x0` 值越小，位置越靠前

## 优化方案

### 核心思路

将当前"先遍历keyword再匹配"的逻辑改为：

1. **收集所有候选**：遍历所有keyword，收集当前页所有实例位置信息
2. **按位置排序**：

   * 正常页面：按 `y0` 从小到大排序（y0小的在前面，即位置靠上）

   * 旋转90度页面：按 `x0` 从小到大排序
3. **逐一匹配pattern**：按排序后的顺序逐一匹配正则pattern
4. **返回首个匹配**：返回第一个匹配成功的信息

### 代码实现

创建一个辅助函数 `collect_and_sort_instances`，统一处理所有关键词搜索逻辑：

```python
def collect_and_sort_instances(page, keywords, pattern, get_search_rect):
    """
    收集所有keyword实例，按位置从上到下排序后匹配pattern，返回第一个匹配结果
    
    PyMuPDF坐标系：原点在左上角，Y轴向下递增
    因此 y0 值越小，位置越靠上
    
    Args:
        page: fitz.Page对象
        keywords: 关键词列表
        pattern: 正则pattern对象
        get_search_rect: 获取搜索区域的函数
    
    Returns:
        匹配成功时返回 {'inst': Rect对象, 'keyword': 关键词字符串}
        未匹配时返回 None
    """
    candidates = []
    
    for keyword in keywords:
        instances = page.search_for(keyword)
        for inst in instances:
            candidates.append({
                'keyword': keyword,
                'inst': inst,
                'y0': inst.y0,
                'x0': inst.x0
            })
    
    if not candidates:
        return None
    
    page_rotation = page.rotation
    if page_rotation == 90:
        candidates.sort(key=lambda c: c['x0'])
    else:
        candidates.sort(key=lambda c: c['y0'])
    
    for candidate in candidates:
        keyword = candidate['keyword']
        inst = candidate['inst']
        rect = get_search_rect(page, inst, keyword)
        text = page.get_text("text", clip=rect)
        if pattern.search(text):
            return {
                'inst': inst,
                'keyword': keyword
            }
    
    return None
```
